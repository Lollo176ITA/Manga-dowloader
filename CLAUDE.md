# Istruzioni di progetto

## Branching

- Le modifiche piccole / fix vanno committate direttamente su `main`.
- Le modifiche grosse ("cicciotte": feature nuove, refactor estesi, lavori multi-file) vanno su un branch `dev` (creato se non esiste) e poi fuse in `main` quando pronte.
- Non creare altri branch di feature senza che l'utente lo chieda esplicitamente.
